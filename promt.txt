baik mari kita lanjutkan..... aku ingin konsepnya seperti kahoot, jadi ada user player dan host, user player dan host memiliki avatar yang unik digenerate dari online website, seperti gambar hewan atau lainnya. Host dapat membuat kuisnya sendiri(choose dan true false). player dapat mengikuti/join kuis dengan kode yang diberikan oleh host, view untuk host dapat melihat nickname dan avatar semua player yang join, view untuk player hanya ada dirinya saja(agar nickname tidak disalahgunakan), kuis dikerjakan berdasarkan waktu, jika waktu habis maka lanjut pertanyaan selanjutnya (setelah next, host dapat melihat statistik jawaban player, misal a 1% b 3% c 6% d 90% dalam kurun waktu 3 atau 5 detik terus lanjut next soal... dan untuk tampilan player setelah next, mungkin cukup menunggu? atau diberi centang jawaban telah terkirim, atau melihat skor terkini), jadi kuis bersifat instant score dan feedback, setiap soal memiliki timer nya masing masing, supaya tidak ada skor ganda, jadi buat kondisi dimana skor di hitung berdasarkan kecepatan menjawab user... dan di bagian akhir juga ada statistik di player ataupun host untuk melihat perkembangan dan kemajuan player... tak lupa di bagian register, buat role untuk host dan player juga.... jadi semua webnya berjalan secara realtime sperti yang di kahoot...... jadi yang kubicarakan diatas itu masuknya ke mode Clasic Mode/Live...... dan untuk satunya mode practice, dimana waktu digunakan untuk per project kuis bukan untuk per soal, jadi misal dikasih waktu 1 menit, maka player harus mengerjakan dalam kurun waktu tersebut, untuk yang sengaja tidak mengerjakan kuis sampai waktu habis, maka otomatis menuju ke halaman hasil dengan skor tertentu/0. Untuk mode ini, player dapat next atau back secara manual, jadi jika soal 1 tidak bisa, bisa dilompati dulu mengerjakan yang lain. taklupa juga untuk menghindari skor ganda, pakai konsep seperti diatas, siapa cepat dapat skor terbaik, jadi waktu berdasarkan waktu submit dan pengerjaan yang benar, dimode ini juga host dapat memantau player nya....Semua mode dapat berhenti secara otomatis/realtime ataupun manual dari host, jika realtime mengikuti timer, jika manual, host harus pencet button untuk stop permainan kuisnya. ini melanjutkan projek ya, ini pakai next js tsx dan supabase....
